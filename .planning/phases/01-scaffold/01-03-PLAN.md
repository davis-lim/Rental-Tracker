Create `server/index.js` with this content:

```js
import express from 'express';
import cors from 'cors';
import db from './db.js';
import router from './routes.js';

const app = express();
const PORT = 3001;

// Middleware
app.use(cors());
app.use(express.json());

// API routes
app.use('/api', router);

app.listen(PORT, () => {
  console.log(`API server running on http://localhost:${PORT}`);
});
```

The `import db from './db.js'` line triggers DB initialization (creates `./data/rental.db` if it doesn't exist). The db variable is not used directly in index.js — route handlers will import it themselves.

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser -> Express API | HTTP requests from React frontend to Express backend (via Vite proxy in dev) |
| Express API -> SQLite | Server-side DB queries |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-05 | Spoofing | CORS | accept | Local-only app, CORS wide-open is fine. No auth needed (single user on localhost). |
| T-01-06 | Injection | server/routes.js | mitigate | All future route handlers must use `db.prepare()` with parameterized queries. Health check has no user input. Placeholder comments in routes.js show the pattern. |
| T-01-07 | Cross-Site Scripting | React app | mitigate | React auto-escapes JSX output by default. No `dangerouslySetInnerHTML` used. Maintain this pattern in future phases. |
| T-01-08 | Denial of Service | Express server | accept | Local-only, not exposed to internet. No rate limiting needed. |
</threat_model>
