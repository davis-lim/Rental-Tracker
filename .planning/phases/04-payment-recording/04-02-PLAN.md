  <action>
Create src/components/RentPaymentFormDialog.jsx. The dialog handles recording, editing, and
clearing a rent payment for a specific tenant+month already chosen on the page.

Props:
  open           boolean
  onOpenChange   fn(bool)
  tenant         { id, name } or null
  month          string YYYY-MM
  existingRecord { id, paid_date, amount_paid, is_on_time } or null
  onSaved        fn(savedRecord)  called after successful upsert
  onDeleted      fn()             called after successful delete

State: paidDate (string), amountPaid (string), error (string|null),
       saving (boolean), showDuplicateConfirm (boolean)

useEffect([open, existingRecord]):
  When open: if existingRecord exists populate paidDate and amountPaid from it,
  else set paidDate to new Date().toISOString().slice(0,10) and amountPaid to empty string.
  Always reset error and showDuplicateConfirm to null/false.

handleSubmit(e):
  1. e.preventDefault()
  2. Validate paidDate with /^\d{4}-\d{2}-\d{2}$/.test(paidDate) — on fail setError and return
  3. Validate parseFloat(amountPaid) > 0 — on fail setError and return
  4. If existingRecord is non-null AND showDuplicateConfirm is false:
       set showDuplicateConfirm(true) and return early — user must submit again to confirm overwrite
  5. setSaving(true)
  6. POST /api/rent-payments with JSON body
       { tenant_id: tenant.id, month, paid_date: paidDate, amount_paid: parseFloat(amountPaid) }
  7. If !res.ok: setError('Could not save. Please try again.'), setSaving(false), return
  8. onSaved(await res.json()), onOpenChange(false)

handleDelete():
  1. fetch DELETE /api/rent-payments/EXISTINGRECORD_ID with method DELETE
  2. If res.ok: onDeleted(), onOpenChange(false)
  3. Else: setError('Could not clear. Please try again.')

Layout using Dialog from @/components/ui/dialog (mirror PropertyFormDialog.jsx):

  DialogContent max-w-[480px] showCloseButton=false
  DialogTitle: existingRecord ? 'Edit Payment' : 'Record Payment'
  Subtitle paragraph: tenant.name then em dash then month, muted-foreground
  paid_date: Label + Input type="text" placeholder="YYYY-MM-DD"
  amount_paid: Label + Input type="number" min="0" step="0.01" placeholder="0.00"

  When showDuplicateConfirm is true render a div with amber/yellow styling above footer:
    bg-amber-50 border border-amber-200 rounded p-3 text-sm text-amber-800
    Text: "A payment for {tenant.name} in {month} already exists. Click 'Overwrite payment' to replace it."

  Error line: p text-sm text-destructive when error is set

  DialogFooter (three buttons, left-aligned delete via mr-auto):
    If existingRecord: Button variant="destructive" onClick=handleDelete "Clear payment" (left side)
    Button variant="outline" onClick=close "Cancel"
    Button type="submit":
      saving => "Saving..."
      showDuplicateConfirm => "Overwrite payment"
      existingRecord => "Update payment"
      else => "Record payment"
  </action>
