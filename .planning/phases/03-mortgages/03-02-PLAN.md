- GET /api/mortgages/:id/dependents → store counts in a local variable

handleDeleteConfirm():
- DELETE /api/mortgages/:id?confirm=true

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| User input → fetch body | Form field values sent as JSON to API |
| API response → React state | Data from /api/mortgages rendered into DOM |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-07 | Spoofing | MortgageFormDialog | mitigate | Client-side validation (lender non-empty, property_id selected, due_day 1-31, amount > 0) before any fetch call; server enforces same rules independently |
| T-03-08 | Tampering | due_day input | mitigate | type="number" min=1 max=31 on input; parseInt validation in onSave before calling API |
| T-03-09 | Information Disclosure | API error display | mitigate | catch blocks show generic "Something went wrong" — no raw API error body or stack traces rendered |
| T-03-10 | Elevation of Privilege | Delete cascade | mitigate | Delete cascade requires explicit user confirmation via AlertDialog — accidental deletions blocked |
| T-03-11 | XSS | React render | accept | React JSX auto-escapes all interpolated values (lender, property_address, etc.) — no dangerouslySetInnerHTML used |
</threat_model>
