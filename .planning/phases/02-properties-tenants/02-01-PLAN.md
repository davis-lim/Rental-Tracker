  <action>
Create `server/services/` directory and `server/services/properties.js` with these exported functions:

1. `getAll()` — `SELECT id, address, notes, created_at, (SELECT COUNT(*) FROM tenants WHERE property_id = properties.id) AS tenant_count FROM properties ORDER BY id DESC`. Returns array.

2. `getById(id)` — `SELECT id, address, notes, created_at, (SELECT COUNT(*) FROM tenants WHERE property_id = properties.id) AS tenant_count FROM properties WHERE id = ?`. Parameter: integer id. Returns object or undefined.

3. `create({ address, notes })` — `INSERT INTO properties (address, notes) VALUES (?, ?)`. Parameters: address (string, required), notes (string, defaults to ''). Returns the created property by calling getById with lastInsertRowid.

4. `update(id, { address, notes })` — `UPDATE properties SET address = ?, notes = ? WHERE id = ?`. Parameters: id (integer), address (string), notes (string). Returns the updated property by calling getById. Returns undefined if id not found (check changes === 0).

5. `remove(id)` — `DELETE FROM properties WHERE id = ?`. Returns `{ deleted: true }` if changes > 0, `{ deleted: false }` if not found.

6. `getDependentCounts(id)` — Returns `{ tenants: N, mortgages: N }` by running:
   - `SELECT COUNT(*) as count FROM tenants WHERE property_id = ?`
   - `SELECT COUNT(*) as count FROM mortgages WHERE property_id = ?`

All queries use `db.prepare(sql).all/get/run(params)` — parameterized (no string interpolation). Import db from `'../db.js'`.

Note: The `remove` function does NOT cascade-delete tenants/mortgages. SQLite foreign keys are ON but without ON DELETE CASCADE in the schema. The route handler will check getDependentCounts first and only delete if the client confirms (sends `?confirm=true`). If dependents exist and confirm is true, delete tenants and mortgages first in a transaction, then delete the property.

Update `remove` to accept a `cascade` boolean parameter:
- `remove(id, cascade = false)`:
  - If cascade is false: plain `DELETE FROM properties WHERE id = ?` (will fail if FK dependents exist with foreign_keys ON)
  - If cascade is true: wrap in `db.transaction(() => { ... })()`:
    1. `DELETE FROM rent_payments WHERE tenant_id IN (SELECT id FROM tenants WHERE property_id = ?)`
    2. `DELETE FROM tenants WHERE property_id = ?`
    3. `DELETE FROM mortgage_payments WHERE mortgage_id IN (SELECT id FROM mortgages WHERE property_id = ?)`
    4. `DELETE FROM mortgages WHERE property_id = ?`
    5. `DELETE FROM properties WHERE id = ?`
  - Returns `{ deleted: true }` or `{ deleted: false }`

**File 1: server/routes/properties.js**

Create Express router with these endpoints:

1. `GET /` — calls `propertiesService.getAll()`, returns 200 with JSON array.

2. `GET /:id` — calls `propertiesService.getById(req.params.id)`. Parse id as integer with `parseInt(req.params.id, 10)`. If NaN, return 400 `{ error: 'Invalid property ID' }`. If not found, return 404 `{ error: 'Property not found' }`. Otherwise 200 with property object.

3. `POST /` — validates request body:
   - `address` is required and must be a non-empty string (after trim). If missing/empty: 400 `{ error: 'Address is required' }`.
   - `notes` defaults to '' if not provided.
   - On success: calls `propertiesService.create({ address: address.trim(), notes: (notes || '').trim() })`, returns 201 with created property.

4. `PUT /:id` — validates:
   - id must be valid integer (400 if NaN)
   - `address` required, non-empty after trim (400 if missing)
   - `notes` defaults to '' if not provided
   - Calls `propertiesService.update(id, { address: address.trim(), notes: (notes || '').trim() })`. If returns undefined: 404. Otherwise 200 with updated property.

5. `GET /:id/dependents` — calls `propertiesService.getDependentCounts(id)`. Returns 200 with `{ tenants: N, mortgages: N }`. This endpoint is called by the UI before showing the delete confirmation dialog. Parse id, validate as integer.

6. `DELETE /:id` — parse id as integer. Check query param `confirm`: if `req.query.confirm === 'true'`, call `propertiesService.remove(id, true)` (cascade). Otherwise call `propertiesService.remove(id, false)`. If the non-cascade delete throws a SQLITE_CONSTRAINT error (because dependents exist), return 409 `{ error: 'Property has dependents', tenants: N, mortgages: N }` (call getDependentCounts to get the numbers). On success: 200 `{ deleted: true }`. On not found: 404.

Wrap each handler body in try/catch. On unexpected errors: 500 `{ error: 'Internal server error' }`.

Import service: `import * as propertiesService from '../services/properties.js';`
Import Router: `import { Router } from 'express';`
Export default the router.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| HTTP request body -> service layer | User-supplied address and notes strings enter SQL queries |
| HTTP URL params -> service layer | :id parameter used in SQL WHERE clauses |
| HTTP query params -> delete logic | ?confirm=true controls cascade behavior |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | Tampering (SQL Injection) | server/services/properties.js | mitigate | All SQL uses db.prepare() with ? placeholders — no string interpolation. Validated in acceptance criteria. |
| T-02-02 | Tampering (Type Coercion) | server/routes/properties.js | mitigate | parseInt(req.params.id, 10) with NaN check returns 400 before any DB call. |
| T-02-03 | Denial of Service | POST /api/properties | accept | Local-only single-user app. No rate limiting needed. Address/notes are unbounded strings but only one user. |
| T-02-04 | Information Disclosure | Error responses | mitigate | Catch blocks return generic "Internal server error" — no stack traces or SQL details leaked. |
| T-02-05 | Elevation of Privilege | DELETE cascade | mitigate | Cascade delete requires explicit ?confirm=true query param. Without it, delete fails with 409 if dependents exist. Transaction ensures atomicity. |
</threat_model>
