CREATE TABLE IF NOT EXISTS tenants (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  property_id INTEGER NOT NULL,
  name TEXT NOT NULL,
  rent_amount REAL NOT NULL,
  deadline_day INTEGER NOT NULL CHECK (deadline_day BETWEEN 1 AND 31),

  created_at TEXT DEFAULT (datetime('now')),

);

CREATE TABLE IF NOT EXISTS rent_payments (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  tenant_id INTEGER NOT NULL,

  paid_date TEXT,
  amount_paid REAL,
  is_on_time INTEGER,
  created_at TEXT DEFAULT (datetime('now')),
  FOREIGN KEY (tenant_id) REFERENCES tenants(id),
  UNIQUE(tenant_id, month)
);

  <action>
Create `server/services/tenants.js` with these exported functions:

1. `getByProperty(propertyId)` — `SELECT id, property_id, name, rent_amount, deadline_day, grace_period_days, lease_start, lease_end, created_at FROM tenants WHERE property_id = ? ORDER BY name ASC`. Parameter: integer. Returns array.

2. `getAll()` — `SELECT t.id, t.property_id, t.name, t.rent_amount, t.deadline_day, t.grace_period_days, t.lease_start, t.lease_end, t.created_at, p.address AS property_address FROM tenants t JOIN properties p ON t.property_id = p.id ORDER BY p.address ASC, t.name ASC`. Returns array with property_address joined.

3. `getById(id)` — `SELECT t.id, t.property_id, t.name, t.rent_amount, t.deadline_day, t.grace_period_days, t.lease_start, t.lease_end, t.created_at, p.address AS property_address FROM tenants t JOIN properties p ON t.property_id = p.id WHERE t.id = ?`. Returns object or undefined.

4. `create({ property_id, name, rent_amount, deadline_day, grace_period_days, lease_start, lease_end })` — INSERT INTO tenants. Parameters:
   - property_id (integer, required)
   - name (string, required)
   - rent_amount (real, required)
   - deadline_day (integer 1-31, required)
   - grace_period_days (integer, default 0)
   - lease_start (string or null)
   - lease_end (string or null)
   SQL: `INSERT INTO tenants (property_id, name, rent_amount, deadline_day, grace_period_days, lease_start, lease_end) VALUES (?, ?, ?, ?, ?, ?, ?)`
   Returns the created tenant by calling getById with lastInsertRowid.

5. `update(id, { name, rent_amount, deadline_day, grace_period_days, lease_start, lease_end })` — `UPDATE tenants SET name = ?, rent_amount = ?, deadline_day = ?, grace_period_days = ?, lease_start = ?, lease_end = ? WHERE id = ?`. Note: property_id is NOT updatable (tenant stays with its property). Returns updated tenant via getById. Returns undefined if id not found (changes === 0).

6. `remove(id, cascade = false)` —
   - If cascade is false: `DELETE FROM tenants WHERE id = ?` (will fail if rent_payments exist with FK on)
   - If cascade is true: `db.transaction(() => { DELETE FROM rent_payments WHERE tenant_id = ?; DELETE FROM tenants WHERE id = ?; })()`
   - Returns `{ deleted: true }` or `{ deleted: false }`

7. `getDependentCounts(id)` — Returns `{ rent_payments: N }`:
   - `SELECT COUNT(*) as count FROM rent_payments WHERE tenant_id = ?`

All queries use `db.prepare(sql)` with `?` placeholders. Import db from `'../db.js'`.

**File 1: server/routes/tenants.js**

Create Express router with these endpoints:

**Property-scoped endpoints** (mounted on the properties router, not this one — see below):
These will be added to `server/routes/properties.js` instead, since they are property-scoped.

Actually, to keep routing clean: add tenant-under-property routes to the properties router file. But to avoid modifying a file owned by 02-01 in this plan, use a different approach:

Mount tenant routes as a separate router. The route structure:
- `GET /` — all tenants globally (for /tenants page per D-07). Calls `tenantService.getAll()`. Returns 200 with array.
- `GET /by-property/:propertyId` — tenants for a specific property. Parse propertyId as int, validate. Calls `tenantService.getByProperty(propertyId)`. Returns 200 with array.
- `GET /:id` — single tenant. Parse id, validate. Calls `tenantService.getById(id)`. 404 if not found.
- `POST /` — create tenant. Validate request body:
  - `property_id` required, must be positive integer. Error: `{ error: 'Valid property_id is required' }`
  - `name` required, non-empty after trim. Error: `{ error: 'Tenant name is required' }`
  - `rent_amount` required, must be a number > 0. Error: `{ error: 'Enter a valid amount greater than 0' }`
  - `deadline_day` required, must be integer 1-28. Error: `{ error: 'Select a due day (1-28)' }` (Note: UI-SPEC says 1-28 in Select, even though schema allows 1-31. Use 1-28 for validation to match UI.)
  - `grace_period_days` optional, defaults to 0, must be >= 0 if provided. Error: `{ error: 'Grace period cannot be negative' }`
  - `lease_start` optional string (date format or null)
  - `lease_end` optional string (date format or null)
  - Verify property_id references an existing property: call propertiesService.getById(property_id). If not found: 400 `{ error: 'Property not found' }`.
  - On success: 201 with created tenant.

- `PUT /:id` — update tenant. Same validation as POST except no property_id change. Parse id. Validate name, rent_amount, deadline_day, grace_period_days (same rules). Call `tenantService.update(id, { ... })`. 404 if not found. 200 on success.

- `GET /:id/dependents` — calls `tenantService.getDependentCounts(id)`. Returns 200 with `{ rent_payments: N }`.

- `DELETE /:id` — parse id. Check query param `confirm`: if `req.query.confirm === 'true'`, call `tenantService.remove(id, true)`. Otherwise call `tenantService.remove(id, false)`. If non-cascade delete throws SQLITE_CONSTRAINT, return 409 `{ error: 'Tenant has payment records', rent_payments: N }` (call getDependentCounts). On success: 200 `{ deleted: true }`. On not found: 404.

Wrap each handler in try/catch. On unexpected errors: 500 `{ error: 'Internal server error' }`.

Import: `import * as tenantService from '../services/tenants.js';`
Import: `import * as propertiesService from '../services/properties.js';` (for property existence check in POST)
